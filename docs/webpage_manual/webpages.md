# GitHubPages にページを追加する方法

## 0. 事前に知っておくこと（超重要）
- 各ページは Markdown（.md）ファイルでできている
- 1ページ = 1つの .md ファイル
- 編集は GitHub の Web 画面だけで可能（PCにソフト不要）
- 間違えても 履歴から必ず戻せる → 安心して作業してよい
- 生成系AIが有能。上手に支持すると完成度の高いmdファイルを作ってくれる。

## 1. 新しいページを作る（基本手順）
### ① リポジトリを開く
- ブラウザで研究室の [GitHub リポジトリ](https://github.com/aiwasaki686/lab-wiki)を開く
- 上部メニューの Code タブにいることを確認
- ユーザーIDとパスワードは別途共有。

### ② ページを置く場所を決める
例：
- NMR 関連 → docs/nmr/
- HPLC 関連 → docs/hplc/
- キラルカラム HPLC → docs/hplc/chiral_column<br>
👉 必ず docs/ フォルダの中に作る。階層構造を意識。

### ③ ページファイル（md ファイル）を作成する
手順１ docs/ フォルダを開く<br>
手順２ 右上の Add file → Create new file をクリック<br>
手順３ ファイル名を入力（例）：<br>
例1 nmr.md　この場合、docsフォルダの中にnmrという名前のmdファイルを作るという意味。<br>
例2 hplc/chiral_column.md　この場合、docsフォルダの中のhplcフォルダ中にchiral_columnというmdファイルを作るという意味。<br>
※ 末尾の拡張子mdを忘れないように。中身が想像できる名前をつけること。

## 2. ページの最初に必ず書く「お約束」
### フロントマター（設定部分）
```
---
title: キラルカラム
parent: HPLC
has_children: true
nav_order: 10
---
```
### 各項目の意味

- title: サイドバーに表示されるページ名（日本語OK）
- parent: 親ページの title 名（完全一致。ファイル名ではない。）　ページをdocs直下に置く場合は不要。階層構造を作る場合のみ記載。
- has_children: true　このページの下に子ページを作る場合に記載する。子ページを作らない場合は不要。
- nav_order: 並び順（小さいほど上）

⚠️ 注意
- : は 半角コロン
- 行頭に 全角スペースを入れない
- いずれの項目も後から簡単に変更できる。

## 3. 本文を書く（Markdownの基本）
### 見出し
```
# 大見出し
## 中見出し
### 小見出し
``` 
### 箇条書き
```
- ポイント1
- ポイント2
```
### 改行
```
<br> を挿入
```
### 太字・強調
```
**重要**
```

### 赤字
```
<span style="color:red;">重要</span>
```

### リンクの貼り方
[]内に表示したい文字列を記載。()内にリンク先のアドレスを記載。
- 本Webページ内でのリンク
```
[NMR](/docs/nmr.md)　階層構造におけるdocs内での位置をカッコ内に記載。
```
- 外部サイト
```
[Bruker公式サイト](https://www.bruker.com/)　アドレスをそのまま貼り付ける。
```

### 画像の貼り方

画像を images/ フォルダにアップロード

本文に以下を書く
```
![説明文](/images/hplc/chiral_example.png)
```

## 4. 保存（Commit）する

画面上の Commit changes をクリック

👉 数分後に Wiki に反映される

## 5. サイドバーに表示されないときのチェック

- title: が書いてあるか

- （入れ子構造としている場合は）parent: が正しいか（親ページの title と一致）

- （入れ子構造としている場合は）親ページに has_children: true があるか

- nav_order が重複していないか


## 6. よくあるエラーと対処
❌ YAML エラーが出る
- 全角コロン（：）を使っていないか
- 行頭に全角スペースが入っていないか
→ 一度消して 英数モードで手打ちすると直ることが多い

❌ 「path does not contain ...」と出る
- リンク先ファイルが存在しない
- フォルダ名・ファイル名のタイプミス
→ Code タブで実際の構造を確認

## 7. 更新のルール
- **個人情報や内部資料は直接書き込まない。**<br>
→ 必要な場合は岩崎研共有フォルダに資料をアップし、リンクを張る。学外者のアクセスが不可となる。<br>
例：<br>
[末端アルキン化合物判定用エクセルシート](https://docs.google.com/spreadsheets/d/1-DKLjEakIeP2p3XObKY4y7g5occmfGF8UX7oBSr5sFc/edit?usp=sharing)<br>
上記リンクのコード<br>
```
[末端アルキン化合物判定用エクセルシート](https://docs.google.com/spreadsheets/d/1-DKLjEakIeP2p3XObKY4y7g5occmfGF8UX7oBSr5sFc/edit?usp=sharing)
```

## 8. 迷ったら

- まず 既存ページを真似る

- 壊してもよい（履歴で戻せる）

- 分からなければ 教員・先輩に聞く

👉 「完成品」を載せる場所ではない。メモ書きレベルでもまず追加することを優先する。
